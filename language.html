Compiler Construction (Racket)

Overview

A semester-long solo project building a compiler incrementally in Racket, targeting a86 — a simplified subset of x86-64 assembly represented as Racket data structures. Each assignment extended the compiler with new language features, from basic primitives all the way to pattern matching and exception handling.



Features Implemented

  • Primitives & Cond — added abs, unary -, not, and cond expressions with type-error signalling
  • Case expressions — dispatch on integer/boolean values, compiled to chains of cmp and je instructions
  • Multi-binding let and let* — generalized let to bind any number of variables simultaneously; let* with forward-referencing bindings
  • List & Vector primitives — heap-allocated values; implemented length, reverse, list->vector, vector->list with type checking
  • Variable-arity & case-lambda — rest arguments collected into lists at runtime; arity dispatch compiled to generated check-and-branch sequences
  • Pattern matching — compiled match expressions for list, vector, and predicate patterns against nested structures
  • Exception handling — implemented raise and with-handlers; used a designated register to hold a pointer to the current handler on the stack, with each handler storing predicate, function, return label, and parent handler pointer for nested handlers.

Every feature added was implemented at two levels: the interpreter (to define correct behavior) and the compiler (to generate correct a86 assembly).



Skills

  • Racket
  • a86 Assembly
  • Compiler Design
  • Stack Management


SmallC Lexer, Parser, and Evaluator (OCaml)

Overview

A solo project implementing a complete compiler frontend and interpreter for SmallC — a statically typed subset of C — in OCaml. The project covered all three stages of language processing: lexing raw source code into tokens, parsing tokens into an abstract syntax tree, and evaluating the AST against a runtime environment. The language supports integers, booleans, arithmetic, conditionals, loops, print statements, and full function definitions with calls.



Lexer

The lexer converts raw SmallC source code into a token list using regular expression matching. Key challenges included longest-match disambiguation — for example, while0 must lex as an identifier not a keyword — and handling negative integer literals correctly. The lexer handles all whitespace, keywords, operators, identifiers, and literals, terminating every output with an EOF token.



Parser

The parser consumes the token stream and produces a typed AST using a hand-written recursive descent parser following a context-free grammar. Operator precedence is encoded directly in the grammar hierarchy — from OrExpr down through AndExpr, EqualityExpr, RelationalExpr, AdditiveExpr, MultiplicativeExpr, PowerExpr, and UnaryExpr. The statement parser builds on top of the expression parser to handle declarations, assignments, if/else, while, for loops, and print. The top-level parser handles full function definitions and enforces that no two parameters share a name and no function is defined twice.



Evaluator

The evaluator walks the AST against an environment represented as a list of variable bindings and function definitions. It enforces static type safety at runtime — raising TypeError on mismatched operands, DivByZeroError on division by zero, and DeclareError on undefined or duplicate variables. Function calls create a fresh environment and binds arguments to parameters while carrying over only the function definitions. Return values are bound to a special "return" identifier in the environment and type-checked against the function's declared return type.



Skills

  • OCaml
  • Lexing
  • Recursive Descent Parsing
  • AST Design


Type Inference Engine for Functional Programming Language (OCaml)

Overview

Built a static type inference engine in OCaml for a custom functional programming language, implementing constraint-based type checking and unification. The project automatically inferred types for expressions without explicit type annotations, supporting integers, booleans, functions, recursive bindings, records, conditionals, and binary operations.



Features Implemented

  • Implemented a constraint generation and unification algorithm to infer types across complex expressions
  • Built a substitution system to propagate inferred types through expressions and resolve temporary type variables
  • Added validation to prevent invalid recursive type constructions during unification
  • Managed scoped environments for variable lookup and function inference
  • Used recursive AST traversal to generate type constraints and perform type resolution.


Skills

  • OCaml
  • Type Systems
  • Abstract Syntax Trees (ASTs)
  • Type Inference


Garbage Collection & Memory Management Simulator (Rust)

Overview

Built a systems-level memory management simulator in Rust implementing multiple garbage collection algorithms, including reference counting, mark-and-sweep, and stop-and-copy collection. The project simulated runtime stack and heap behavior, tracking object references, reachability, allocation, deallocation, and heap compaction. It modeled how programming language runtimes manage dynamic memory and reclaim unreachable objects while maintaining pointer consistency across memory regions.



Features Implemented

  • Implemented a reference counting memory manager that tracked heap object references during stack pushes, heap allocations, and object deallocation
  • Added recursive cleanup logic to free chains of unreachable objects when reference counts dropped to zero
  • Built a reachability analysis algorithm using graph traversal to identify all heap objects accessible from stack references
  • Implemented a mark-and-sweep garbage collector that reclaimed unreachable heap memory after traversal
  • Developed a stop-and-copy collector that copied live objects between heap regions and compacted memory
  • Updated all stack and heap pointers after copying to maintain correct object references
  • Simulated stack frames and heap memory using custom Rust data structures
  • Applied Rust’s ownership and borrowing system to safely manage mutable heap structures and references


Skills

  • Rust
  • Garbage Collection Algorithms
  • Systems Programming
  • Heap & Stack Memory Management
  • Reference Counting
  • Mark-and-Sweep Collection
  • Stop-and-Copy Collection
